# News Topic Classification Pipeline

## 1. Описание задачи и датасета
Проект реализует мультиагентный дата-пайплайн для **многоклассовой классификации новостных текстов**.  
Цель — собрать данные из нескольких источников, очистить их, автоматически разметить, передать неуверенные примеры на ручную проверку и обучить итоговую модель классификации.

### Классы
- `world`
- `sports`
- `business`
- `sci_tech`

### Источники данных
- открытый датасет из HuggingFace (`ag_news`)
- RSS-источники новостей

### Унифицированная схема
Все данные приводятся к общей структуре:
- `text`
- `label`
- `source`
- `collected_at`

### Размер датасета
- всего собрано: **2126** записей
- дубликатов по тексту до чистки: **14**

## 2. Что делал каждый агент

### DataCollectionAgent
Собирал новости из HuggingFace и RSS-источников, затем объединял их в единый DataFrame со стандартной схемой.

### DataQualityAgent
Проверял:
- пропуски
- дубликаты
- выбросы по длине текста
- дисбаланс классов

Базовая стратегия чистки:
```python
strategy = {
    "missing": "drop",
    "duplicates": "drop",
    "outliers": "clip_iqr"
}
```

### AnnotationAgent
Выполнял авторазметку текстов по 4 темам, вычислял confidence и экспортировал примеры с низкой уверенностью в `review_queue.csv`.  
Также был подготовлен файл спецификации разметки с описанием классов, пограничных случаев и примерами.

### ActiveLearningAgent / финальный этап обучения
На итоговых данных использовалась baseline-модель:
- `TfidfVectorizer`
- `LogisticRegression`

Модель сохранена в `models/final_model.joblib`.

## 3. Human-in-the-loop
Точка ручной проверки реализована через файл `review_queue.csv`.

В этот файл попадают примеры с низкой уверенностью авторазметки.  
Далее человек:
1. открывает CSV,
2. проверяет спорные тексты,
3. исправляет метки,
4. возвращает файл для повторного использования в пайплайне.

### Фактический объём ручной проверки
- размер review queue: **513** примеров

## 4. Метрики качества

### Качество авторазметки
- Accuracy: **0.4858**
- Macro F1: **0.4849**
- Cohen's kappa: **0.2782**

### Качество итоговой модели
- Accuracy: **0.9167**
- Macro F1: **0.9149**

## 5. Ретроспектива

### Что сработало хорошо
- единая схема данных для разных источников;
- корректная выгрузка review queue для human-in-the-loop;
- хорошее качество итоговой TF-IDF + Logistic Regression модели.

### Что можно улучшить
- повысить качество авторазметки;
- уменьшить дисбаланс классов;
- сохранять полную историю Active Learning по итерациям;
- заменить простую авторазметку на более сильную zero-shot или LLM-based схему.
